EDGE CASE PROBES
=======================================================================

1. Single-waypoint FIRST stage (e2e-stage-1 in test data):
   Code reference: frontend/src/lib/utils/fullProfile.ts:83-113
   Behavior: xStart=xEnd=0 because prevWp=null for the first waypoint ever (no prior point).
   Hit area width = Math.max(1, scaleX(0)-scaleX(0)) = 1 SVG unit (1/1000 of width).
   This is functionally a near-invisible hit area for stage1 in the test data.
   Assessment: NOT a spec violation. AC-11 only discusses stages WITHOUT waypoints.
   The spec's §3 hit-area description assumes stages have multiple waypoints.
   Severity: LOW (UX note, no AC violated, not a blocker).
   No E2E test clicks the SVG hit area of stage1, so no test breakage.

2. Pill tone mismatch (spec says 'neutral', code uses 'default'):
   Code reference: frontend/src/lib/components/trip-detail/StageDetailRow.svelte:57
   Spec reference: docs/specs/modules/epic_135_step4_left_column.md §5 (line 300)
   Spec says: tone={active ? 'accent' : 'neutral'}
   Implementation uses: tone={active ? 'accent' : 'default'}
   Pill interface: frontend/src/lib/components/ui/pill/Pill.svelte:7
   Pill only supports: 'default' | 'success' | 'warning' | 'danger' | 'info' | 'accent'
   'neutral' is NOT in the Pill interface. The implementation is CORRECT; the spec has a typo.
   Severity: LOW (spec typo, implementation handles correctly).

3. Safari pattern compliance:
   Code reference: frontend/src/lib/components/trip-detail/FullProfile.svelte:101-105
   Code reference: frontend/src/lib/components/trip-detail/StageList.svelte:27-31
   Both components use factory pattern (makeStageClickHandler, makeSelectHandler).
   StageDetailRow uses native onclick={onSelect} where onSelect is pre-bound — also Safari-safe.
   Assessment: FULLY COMPLIANT with Safari requirements.

4. Date/timezone consistency:
   Code reference: frontend/src/lib/utils/fullProfile.ts:134-138 (getActiveStageId uses local time)
   Code reference: frontend/src/lib/utils/tripHero.ts:39-43 (todayIso uses local time — identical pattern)
   Code reference: frontend/src/lib/utils/tripStatus.ts:29 (deriveTripStatus uses UTC)
   Assessment: getActiveStageId matches tripHero.ts exactly (both local time). Pre-existing design
   choice documented in spec Known Limitations. Not a regression introduced by Step 4.

5. Backend empty stages authorization:
   Code reference: internal/handler/trip.go:63-66
   Comment in code explains PO authorization and reason (AC-15, Wizard-Drafts).
   validateTrip still checks id, name, and per-waypoint coordinates. Logic is consistent.
   Assessment: Authorized change, no regression risk to existing trips.

6. computeHeaderStats with null elevation in waypoints (NaN risk):
   Code reference: frontend/src/lib/components/email-preview/headerStats.ts:54
   For stages with null elevation_m, Math.round(NaN) = NaN would appear in stat-strip.
