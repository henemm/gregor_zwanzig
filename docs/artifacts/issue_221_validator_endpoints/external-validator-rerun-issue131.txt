### Empfehlung (gemäß Memory-Regel „Bei AMBIGUOUS Sichtbarkeit nachrüsten")

Damit eine erneute External-Validierung möglich wird, mindestens **einen** dieser Endpoints einbauen:

1. `GET /api/validator/format-metric-value?unit=m&value=12240` → `{"formatted":"12.240 m"}` (deckt AC-4/5/6).
2. `POST /api/validator/alert-render` mit Trip-ID + synthetischen Changes → `{"html":"…","plain":"…"}` (deckt AC-1/7/8).
3. `GET /api/validator/alert-detector/thresholds?trip_id=<id>` → Threshold-Map + verwendeter Pfad (`from_display_config`/`from_trip_config`) (deckt AC-2/3).

Alternativ ist die Spec bewusst als „nur durch pytest verifizierbar" zu markieren — dann liegt die Verifikation außerhalb der External-Validator-Reichweite.
