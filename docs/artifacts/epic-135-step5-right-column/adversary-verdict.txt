ADVERSARY FINDINGS

Finding F001
  Severity: LOW
  Category: anti_pattern
  Description: `prettyLabel` is exported from rightColumn.ts and is the 5th
    public function in the module, but it has zero direct unit tests. The
    import statement in rightColumn.test.ts does not import `prettyLabel` at
    all; it only appears in a comment. The E2E suite checks chip visibility by
    testid but never asserts the human-readable label text (e.g. that
    `temp_min` yields "Min-Temp", or that unknown keys fall back to the key
    itself).
  Code reference: /home/hem/gregor_zwanzig/frontend/src/lib/utils/rightColumn.test.ts:19
    (import statement omits prettyLabel)
  Code reference: /home/hem/gregor_zwanzig/frontend/src/lib/utils/rightColumn.ts:78
    (prettyLabel implementation)
  Evidence: `grep -n "prettyLabel" rightColumn.test.ts` returns only a comment
    on line 8. No describe block for prettyLabel exists.
  Spec reference: §1 (prettyLabel listed as 5th exported function), spec says
    "mind. 12" unit tests — that bar is cleared (25 tests), but the 5th
    function is untested.
  Remediation: Add a `describe('prettyLabel', ...)` block to rightColumn.test.ts
    covering: known key (temp_min → 'Min-Temp'), unknown key falls back to key
    itself. 2–3 tests sufficient.

Finding F002
  Severity: LOW
  Category: anti_pattern (test hygiene)
  Description: The AC-3 E2E test creates a trip `e2e-no-briefing-trip` via POST
    and never deletes it. There is no afterEach/afterAll cleanup for this
    temporary trip. Across repeated test runs the trip accumulates in the test
    database, which may cause ID conflicts or clutter.
  Code reference: /home/hem/gregor_zwanzig/frontend/e2e/trip-detail-overview-right.spec.ts:85-116
  Evidence: No DELETE /api/trips/e2e-no-briefing-trip call exists anywhere in
    the spec file; afterAll only calls resetTripState(e2e-cockpit-test).
  Remediation: Add cleanup in afterAll:
    await request.delete(`/api/trips/${NO_BRIEFING_TRIP_ID}`);
  Note: Pre-existing pattern in trip-detail-overview-left.spec.ts (AC-15 also
    creates a temporary trip without cleanup), so this is a project-wide
    pattern, not unique to Step 5.

Finding F003
  Severity: LOW
  Category: anti_pattern (a11y)
  Description: All 4 cards render as plain `<div>` (via GCard), without
    role="region" or aria-label. The `<h3>` headings inside them have no
    enclosing landmark. The `<aside>` container (`trip-overview-right-column`)
    has no aria-label either.
  Code reference: /home/hem/gregor_zwanzig/frontend/src/lib/components/ui/g-card/GCard.svelte:18
  Code reference: /home/hem/gregor_zwanzig/frontend/src/lib/components/trip-detail/TripOverview.svelte:52
  Evidence: GCard renders <div data-slot="g-card">. No ARIA landmark or label
    on the aside or any card.
  Spec reference: Spec does NOT require <section> or aria-label — this is an
    advisory finding only, not a spec violation.
  Remediation: Low priority; can be addressed in a future accessibility pass.
    Adding aria-label="Briefings" / aria-label="Wetter-Metriken" etc. to each
    GCard would improve screen-reader experience.
