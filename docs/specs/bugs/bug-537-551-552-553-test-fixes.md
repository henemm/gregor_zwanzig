## Implementation Details

### Fix #537 — Zeitbombe in `_create_workflow()`

`tests/tdd/test_issue_258_hook_arch.py:112-113, 132`: Hardcoded Datum `"2026-05-18T10:00:00"` → seit 2026-06-01 ≥14 Tage alt → `_sweep_candidates()` klassifiziert phase1-Workflows fälschlich als stale.

```python
# Zeile 112-113 (IST):
"created": "2026-05-18T10:00:00",
"last_updated": "2026-05-18T10:00:00",
# Zeile 132 (IST):
log_path = log_dir / f"2026-05-18_{name}.yaml"

# SOLL (alle drei Stellen):
from datetime import datetime
now_str = datetime.now().isoformat()
"created": now_str,
"last_updated": now_str,
log_path = log_dir / f"{datetime.now().strftime('%Y-%m-%d')}_{name}.yaml"
```

### Fix #551 — Keyword-only-Argument in `build_segment_label`

`tests/tdd/test_issue_397_segment_timezone.py:115`:

```python
# IST (falsch — positionales Arg):
label = build_segment_label(change, [seg_data], _CEST)

# SOLL:
label = build_segment_label(change, [seg_data], tz=_CEST)
```

### Fix #552 — Obsolete Tests durch #496-Redesign

`tests/tdd/test_issue_376_channel_preview_select.py`: Alle 4 Tests prüfen ob `ChannelPreviewBlock.svelte` die `Select`-Komponente aus `$lib/components/ui/select` nutzt. Nach dem #496-Redesign (tab-basierte Kanalauswahl) ist diese Anforderung intentional weggefallen. Die Testdatei wird gelöscht.

### Fix #553 — Purge-Test an neue Semantik anpassen

`tests/tdd/test_alert_log.py:95-135`: `test_append_alert_log_purges_entries_older_than_48h` erwartet `len == 2` (alter Eintrag entfernt). Seit #396 werden alle Einträge dauerhaft behalten (historische Statistik). Test wird auf aktuelle Semantik umgeschrieben: alle 3 Einträge (alt, frisch, neu) bleiben erhalten.

```python
# IST (falsch):
assert len(data["entries"]) == 2
severities = [e["severity"] for e in data["entries"]]
assert "LOW" not in severities

# SOLL:
assert len(data["entries"]) == 3  # kein Purge mehr — alle bleiben
severities = [e["severity"] for e in data["entries"]]
assert "LOW" in severities    # alter Eintrag bleibt erhalten
assert "MODERATE" in severities
assert "HIGH" in severities
```
