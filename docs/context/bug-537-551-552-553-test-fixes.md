## Diagnose je Issue

### #537 — Zeitbombe
`_create_workflow()` setzt `created` und `last_updated` auf `"2026-05-18T10:00:00"`.
`_sweep_candidates()` klassifiziert Workflows als stale wenn `age >= 14 Tage`.
Ab 2026-06-01 (>14 Tage) werden phase1-Workflows in den Tests fälschlich als stale erkannt.
**Fix:** `datetime.now().isoformat()` statt hartem Datum. Auch `log_path`-Name (Zeile 132) dynamisch.

### #551 — Falsches Argument
```python
# IST (falsch):
build_segment_label(change, [seg_data], _CEST)
# SOLL:
build_segment_label(change, [seg_data], tz=_CEST)
```

### #552 — Obsolete Tests durch #496-Redesign
`ChannelPreviewBlock.svelte` hat nach #496 keine Select.svelte-Komponente mehr.
Aktuelle Architektur: Tab-basierte Kanalauswahl mit `onSelect`-Callback.
Tests prüfen eine Anforderung, die intentional aufgegeben wurde.
**Fix:** Gesamte Testdatei löschen, da alle 4 ACs obsolet sind.

### #553 — Intentionale Semantikänderung
`_append_alert_log` purgt seit #396 keine alten Einträge mehr (historische Stats).
Test erwartet `len == 2` (alt entfernt + frisch + neu), bekommt 3 (alle).
**Fix:** Test auf aktuelle Semantik anpassen (kein Purge, alle Einträge bleiben).
