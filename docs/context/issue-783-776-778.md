# Context: issue-783-776-778

## Request Summary
Drei E-Mail-Briefing-Bugs: (#783) Briefing-Stundentabelle ignoriert geänderte Trip-Startzeit;
(#776) E-Mail-Inhalts-Sektionen reagieren nicht auf Frontend-Toggles; (#778) toter Code in trip_report.py.

## Root Causes (verifiziert)

### #783 — Startzeit ignoriert (backend, Python)
- `extract_hourly_rows` (src/output/renderers/email/helpers.py:123-137) fenstert die Stundentabelle
  auf `seg_data.segment.start_time.hour`.
- `_convert_trip_to_segments` (src/services/trip_report_scheduler.py:686-815): für den ERSTEN
  Waypoint (i==0) hat `wp1.arrival_calculated`/`arrival_override` (persistierter Naismith-Wert,
  Z.728-738) VORRANG vor `default_start = stage.start_time` (Z.714). Ändert der Nutzer die
  Stage-Startzeit auf 14:00, bleibt der stale Waypoint-Arrival (z.B. 07:00) Quelle des Fensters.
- Stage.start_time wird korrekt persistiert (Go internal/model/trip.go:85, Python loader) — es wird
  nur in der Segmenterzeugung nicht respektiert.

### #776 — Sektionen-Toggles ohne Wirkung (frontend, Svelte)
- Voller Render-Pfad konsumiert report_config korrekt: trip_report.py:124-171 → render_email →
  html.py-Gates (335 stage_stats, 622/734 metrics_summary, 755 outlook). Save/Load der anderen Tabs ok.
- BRUCH: `WeatherMetricsTab.svelte` zeigt die E-Mail-Inhalt-Karte (Z.530 `showMailContent={true}`),
  aber `handleSave()` (Z.347-360) PUTet nur `display_config` an `/api/trips/{id}/weather-config` —
  das gebundene `reportConfig` wird NIE gesendet. Zudem ignoriert `isDirty` (Z.136) reportConfig →
  Toggle markiert nicht mal dirty. Die Inhalts-Bausteine in diesem Tab persistieren nie.
- Korrekt speichernde Tabs zum Vergleich: BriefingsTab (Z.20), BriefingScheduleTab (Z.48).
- Nebenbefund: show_quick_take_tags/show_stability/show_highlights/show_yesterday_comparison haben
  GAR KEINE UI-Checkbox (nur $state-Default true) → können nie ausgeschaltet werden.

### #778 — Toter Code (backend, Python, Tech-Debt)
- src/formatters/trip_report.py: `_fmt_val` (~691), `_render_html_table` (928), `_render_text_table`
  (953), `_format_daylight_html/plain` (833/888) sind tot — format_email delegiert seit β3 an
  render_email(); kanonische Logik in src/output/renderers/email/. _fmt_val liest stale `blue`-Key.
