Lies die Datei `claude-code-handoff/INSTRUCTIONS.md` in diesem Repo und führe
den dort beschriebenen Plan aus:

1. Manifest prüfen — `bash claude-code-handoff/check-manifest.sh`. Bei
   Mismatch sofort stoppen und fresh zip anfordern.
2. `gh auth status` verifizieren.
3. Fehlende Labels anlegen (Liste in INSTRUCTIONS.md, inkl. `area:output`
   und neu: `accessibility`).
4. Screenshots aus `claude-code-handoff/screenshots/` nach
   `.github/issue-assets/` committen und nach `main` pushen.
5. **Dedupe vor Anlegen** — für jeden Eintrag aus `issues.json`:
     - GitHub durchsuchen nach
       `in:body gregor-zwanzig-handoff: stable_id=<id>`
     - Closed-Match → SKIP (nichts tun)
     - Open-Match → UPDATE body + ggf. fehlende Labels (Titel unverändert)
     - Kein Match + status="new" → CREATE
     - Kein Match + status="existing-or-update" → FRAG vor Anlegen (kann
       absichtlich gelöscht worden sein)
6. Am Ende pro-Issue-Zusammenfassung drucken:
   CREATED / UPDATED / SKIPPED-CLOSED / ASKED, jeweils mit URL.
